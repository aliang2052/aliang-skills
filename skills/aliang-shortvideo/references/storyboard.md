# 分镜规范（枢纽）

分镜是整条流水线的**枢纽**：它同时承载「画面 prompt（喂图片/视频）+ 时长 + 运镜 + 台词」。上游剧本拆进来，下游图片/视频从这里取料。产出**两份同源数据**：给人看的表格 + 给脚本用的 `storyboard.json`。

## 一、分镜表格（给人看）

竖版 9:16 默认。每镜一行：

| 镜号 | 时长 | 景别 | 运镜 | 画面内容 | 角色/动作 | 台词/旁白 | 情绪 |
|------|------|------|------|----------|-----------|-----------|------|
| 1 | 5s | 特写 | 推 | 雨夜，外卖箱上的水珠 | 小林低头攥紧订单 | （旁白）他已经三天没接到单 | 压抑 |
| 2 | 5s | 中景 | 固定 | 老旧居民楼门口 | 小林按门铃，门开，怔住 | —— | 错愕 |

镜数估算：总时长 ÷ 单镜均长（4-6s）。例：30s≈6 镜，60s≈12 镜。

## 二、storyboard.json（给脚本用）

`scripts/gen_images.py` 读这个文件批量出图。结构：

```json
{
  "title": "雨夜送单",
  "genre": "草根逆袭",
  "aspect": "9:16",
  "style": "电影感写实，冷色调，胶片颗粒，夜景霓虹",
  "style_ref": null,
  "negative_prompt": null,
  "characters": [
    {"name": "小林", "desc": "25岁男性，瘦削，短发，灰色外卖服，眉眼疲惫但坚定", "ref": null}
  ],
  "shots": [
    {
      "id": 1,
      "duration": 5,
      "shot_size": "特写",
      "camera": "推",
      "image_prompt": "雨夜，外卖保温箱表面密布水珠，霓虹倒影，浅景深",
      "characters": [],
      "dialogue": "（旁白）他已经三天没接到单了",
      "emotion": "压抑"
    },
    {
      "id": 2,
      "duration": 5,
      "shot_size": "中景",
      "camera": "固定",
      "image_prompt": "老旧居民楼楼道门口，昏黄灯光，小林站在门前怔住",
      "characters": ["小林"],
      "dialogue": "",
      "emotion": "错愕"
    }
  ]
}
```

字段说明：
- `style`：全片统一画风，**每张图都会自动拼上**，是一致性的核心。从 `references/styles.md` 选预设填。
- `style_ref`：可选，**全局风格参考图**路径（用户甩的海报/截图）。无角色 ref 的镜头会用 `bl image edit` 迁移这张图的画风。
- `negative_prompt`：可选，覆盖默认负向词（脚本已内置一套防瑕疵默认值，一般留 null）。
- `characters[].desc`：人物固定外貌描述，出现该角色的镜头自动拼接，保证长相统一。
- `characters[].ref`：可选，角色基准图路径；填了就用 `bl image edit` 带参考图保持长相（更稳）。
- `shots[].image_prompt`：**只写画面**，不要写台词/镜号/时长（那些是给视频和剪辑的）。
- `shots[].characters`：本镜出现的角色名（用于拼接角色外貌描述）。
- `dialogue`/`duration`/`camera`：图片阶段用不到，但要保留，供后续视频/剪辑阶段使用。

## 三、时长与片段规则

时长**两个层级都由用户掌控**，skill 默认值只在用户没指定时兜底：

1. **总时长**（开工时确认）：决定镜数。速算 **镜数 ≈ 总时长 ÷ 5**（15s≈3 镜、30s≈6 镜、60s≈12 镜）。
2. **单镜时长**（分镜阶段逐镜可调）：默认 5s，用户可把某镜设 10s / 15s。

⚠️ 图生视频模型单段通常 **5s 为基本单位**（`bl video generate --duration` 实测 2-10s）。所以「10s 镜 = 2 个 5s 片段」「15s 镜 = 3 段」，在分镜阶段就要把长镜**拆成多个 5s 子镜**（同画面延续或递进），否则视频阶段无法一次产出。

- 图片阶段：每个（子）镜出 1 张关键帧。**加长某镜 = 该镜图片数增加**。
- 拆分后把「总时长 / 总镜数 / 总张数」报给用户对一下账。
- 默认竖版 9:16（抖音/快手/视频号）；横版 16:9 需用户指定。

## 四、画面 prompt 写法

好的 image_prompt = **主体 + 环境 + 光线 + 构图/景别**，中文即可（Qwen-Image 中文友好）：

- ✅「雨夜便利店门口，暖黄灯光，男青年蹲在台阶上，中景，浅景深，电影感」
- ❌「小林很难过」（太抽象，没画面）
- 不要把对话、字幕、镜号写进 prompt。
- 情绪靠光线/环境/姿态表达，不靠文字标注。

## 五、景别与运镜词汇

**景别**：远景、全景、中景、近景、特写、大特写。
**运镜**：固定、推、拉、摇、移、跟、升、降、环绕。
（运镜词主要写给后续视频阶段；图片阶段景别会影响构图，可在 image_prompt 末尾点出"中景/特写"。）

## 六、画风与角色一致性锚点

短剧成图最容易翻车的就是**画风跳 + 人物长相不一致**。锚点：

1. **统一 style 串**：`storyboard.json` 的 `style` 字段，每张图自动拼接，全片一套（从 styles.md 选预设）。
2. **固定角色描述**：`characters[].desc` 写死外貌（年龄/发型/服装/特征），出现即拼接。
3. **固定随机种子**：同一角色尽量复用同一 seed，脚本按角色名稳定偏移自动处理。
4. **参考图（最稳，gen_images.py 已支持）**——优先级：**角色 ref > 全局 style_ref > 纯文生图**：
   - `characters[].ref`：主角基准图，锁脸/锁服装。先出一张满意的主角图，填回 `ref`，后续镜头自动切 `bl image edit` 带图微调场景。
   - 顶层 `style_ref`：用户甩的一张参考海报/截图，锁全片画风。无角色 ref 的镜头用它做 `bl image edit` 风格迁移。
   - 两者可并用：有角色 ref 的镜头锁脸，其余镜头用 style_ref 锁画风。
